## 한 줄 정의
데이터 자체가 아니라 데이터의 생성 시점, 출처, 작성자, 버전, 분류 등을 설명하는 "데이터에 관한 데이터"이다.

## 왜 중요한가(실무)
AI 시대에 데이터는 원유에 비유되지만, 메타데이터 없는 데이터는 라벨 없는 원유 드럼통과 같다. 어디서 채굴했는지, 품질은 어떤지, 유통기한은 언제까지인지 알 수 없으면 활용할 수 없다. 조직에 데이터가 아무리 많아도 메타데이터가 정리되지 않으면 "어떤 데이터가 어디에 있는지" 파악하는 것조차 불가능하다.

AI 모델 학습에서 메타데이터의 역할은 더욱 커진다. 학습 데이터의 수집 시점, 출처, 라벨링 담당자, 전처리 이력 등이 기록되어야 모델의 편향이나 품질 문제를 진단할 수 있다. 예를 들어, 특정 기간 데이터만으로 학습한 모델이 최신 트렌드를 반영하지 못할 때, 메타데이터를 통해 학습 데이터의 시간 범위를 확인하고 보완할 수 있다.

데이터 거버넌스 관점에서도 메타데이터는 핵심이다. 개인정보가 포함된 데이터인지, 외부 공유가 허용된 데이터인지, 보존 기한이 언제까지인지 등의 정보가 메타데이터로 관리되어야 규정 준수가 가능하다. 메타데이터 관리 없이 데이터 거버넌스를 시행하는 것은 사실상 불가능하다.

## 핵심 이론(직관)
### 1) 메타데이터의 세 가지 유형
기술적 메타데이터(technical metadata)는 파일 형식, 크기, 스키마 등 구조 정보를 담는다. 비즈니스 메타데이터(business metadata)는 데이터의 의미, 소유자, 활용 목적 등 맥락 정보를 담는다. 운영 메타데이터(operational metadata)는 생성 일시, 접근 빈도, 처리 이력 등 활용 정보를 담는다. 세 유형이 모두 갖춰져야 데이터를 제대로 관리할 수 있다.

### 2) 데이터 리니지(Data Lineage)와의 관계
메타데이터 중에서도 데이터의 이동 경로와 변환 이력을 추적하는 것을 데이터 리니지라 한다. 원천 시스템에서 추출되어 어떤 변환을 거쳐 최종 분석 테이블에 도달했는지를 보여준다. AI 파이프라인에서는 원본 데이터 → 전처리 → 피처 엔지니어링 → 모델 입력까지의 전 과정이 리니지로 관리되어야 한다.

### 3) 메타데이터 자동 수집과 카탈로그
메타데이터를 수동으로 입력·관리하면 금방 낙후된다. 현대적인 접근은 데이터 카탈로그 도구를 활용하여 기술적 메타데이터를 자동 수집하고, 비즈니스 메타데이터는 데이터 소유자가 등록·갱신하는 방식이다. 검색 가능한 데이터 카탈로그가 있으면 조직 내 누구나 필요한 데이터를 빠르게 찾을 수 있다.

## 실무 포인트
### 1) AI 프로젝트에서의 메타데이터 관리
모델 학습에 사용된 데이터셋마다 수집 기간, 출처, 샘플 수, 라벨링 방법, 전처리 내역을 기록해야 한다. 모델 자체에도 버전, 학습 하이퍼파라미터, 성능 지표, 배포 일시 등의 메타데이터를 부여한다. 이 정보가 없으면 모델 재현이나 비교가 불가능해진다.

### 2) 조직 차원의 메타데이터 표준 수립
부서마다 "고객"의 정의가 다르고, 날짜 형식이 다르면 데이터 통합이 불가능하다. 메타데이터 표준(필드명 규칙, 분류 체계, 필수 입력 항목 등)을 조직 차원에서 정하고, 데이터 사전과 연동하여 일관성을 확보해야 한다.

### 3) 개인정보 분류 태깅
개인정보보호법과 AI 규제 대응을 위해 데이터셋에 민감도 등급(공개/내부/기밀/민감개인정보)을 메타데이터로 태깅하는 것이 필수다. 이 태깅이 있어야 AI 학습 데이터로 사용 가능한지, 외부 API에 전송 가능한지를 자동으로 판단할 수 있다.

## 체크리스트
- 주요 데이터셋에 생성 시점, 출처, 소유자, 갱신 주기가 기록되어 있는가
- AI 모델 학습에 사용된 데이터의 버전과 전처리 이력이 추적 가능한가
- 데이터 민감도 등급(개인정보 포함 여부)이 메타데이터로 분류되어 있는가
- 데이터 리니지를 파악할 수 있는 체계가 있는가
- 메타데이터 입력 표준(필드명, 형식, 필수 항목)이 정의되어 있는가
- 데이터 카탈로그 또는 이에 준하는 검색 가능한 메타데이터 저장소가 있는가
- 메타데이터가 주기적으로 갱신되고, 낙후된 정보를 탐지하는 프로세스가 있는가
