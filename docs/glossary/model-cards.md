## 한 줄 정의
모델 카드(Model Card)는 모델의 목적, 사용 범위, 평가, 한계, 위험을 문서로 정리한 형식이다.

## 왜 중요한가(실무)
모델이 ‘어디까지 믿어도 되는지’가 불명확하면 과신(Overreliance)이 생긴다.  
모델 카드는 최소한의 공통 언어를 만든다.
- 어떤 업무에 쓰면 안 되는가
- 어떤 데이터는 넣으면 안 되는가
- 어떤 테스트를 통과했는가
- 어떤 실패 패턴이 알려져 있는가

## 모델 카드에 들어가야 할 항목(실무형)
- 목적/사용 시나리오(허용/금지)
- 입력 데이터 가이드(민감정보, 허용 포맷)
- 평가 결과(골든 세트, 정확도/오류 유형/리스크)
- 한계(환각, 숫자 오류, 최신성, 컨텍스트 한계)
- 안전/보안(인젝션, 출력 처리, 로그 정책)
- 변경 이력(버전, 날짜, 변경 내용)

## 실무 팁
- 처음부터 완벽히 쓰지 말고 1페이지로 시작해도 된다.
- 모델 카드가 있으면 벤더 변경/모델 변경 시 비교 기준이 생긴다.
- 에이전트형/툴 호출 시스템은 “시스템 카드” 형태로 확장하는 것이 좋다(툴/권한/승인/로그 포함).
