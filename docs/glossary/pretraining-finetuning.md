## 한 줄 정의
- **사전학습(Pretraining)**: 대규모 일반 데이터로 기본 언어 능력/지식을 학습하는 단계
- **미세조정(Fine-tuning)**: 특정 목적/도메인/스타일에 맞게 추가 학습하는 단계

## 왜 중요한가(실무)
실무에서는 “어떤 모델을 쓸까”보다 “어떤 방식으로 품질을 올릴까”가 더 중요해진다.  
그 선택지는 대략 아래로 나뉜다.
- 프롬프트/템플릿 개선(가장 빠르고 안전한 편)
- RAG로 근거를 붙임(지식 최신성/근거 통제)
- 미세조정으로 특정 작업 성능/스타일을 고정(하지만 데이터/운영 부담)

## 핵심 이론(필요한 만큼)
### 1) 사전학습은 “범용 능력”을 만든다
사전학습은 엄청난 양의 텍스트 패턴을 학습해, 문장 생성/요약/추론 같은 기본 능력을 만든다.

### 2) 미세조정은 “특정 목적”을 강화한다
예: 우리 조직의 문체, 특정 양식(보고서 템플릿), 특정 분류 작업 등  
단, 미세조정은 아래 위험이 뒤따른다.
- 학습 데이터에 민감정보/저작권 문제가 섞일 수 있음
- 데이터 품질이 낮으면 오히려 성능이 나빠질 수 있음
- 운영(평가/버전 관리/재학습) 비용이 발생

## 실무 의사결정 가이드
### 미세조정이 유리한 경우
- 출력 형식/스타일을 매우 강하게 고정해야 함
- 반복 작업이 많고, 프롬프트만으로는 변동이 큼

### RAG가 유리한 경우
- 최신 정책/문서가 자주 바뀜
- “근거를 어디서 가져왔는지”가 중요함

### 프롬프트/템플릿이 먼저인 경우
- 운영을 빠르게 시작해야 함
- 리스크(대외/규정)가 커서 변경을 통제해야 함

## 체크리스트
- 학습에 사용할 데이터에 민감정보/저작권 이슈가 없는가
- 골든 세트로 회귀 테스트가 준비되어 있는가
- 미세조정 후 롤백/버전 관리가 가능한가
