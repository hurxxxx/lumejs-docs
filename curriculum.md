# AI Group Study Curriculum (6주)

- 버전: 2026-02-03
- 대상: AI/코딩 경험이 없는 비전공자
- 목표: **AI 기초 → 실무 적용 → AX 컨설팅 제안**까지 단계 학습

---

## 진행 원칙
- **초보 친화**: 용어를 쉬운 언어로 설명하고, 반복 학습 구조를 유지
- **이론–실습 연결**: 핵심 개념을 바로 실습으로 연결해 체득
- **아키텍처/설계 관점**: 프로세스와 솔루션 구조를 큰 그림으로 이해
- **실습 중심**: 매 회차 최소 50%는 직접 실습
- **현업 맥락**: 실제 업무/생활 문제를 기준으로 학습
- **안전/윤리**: 민감정보 입력 금지, 결과 검증 필수

## 세션 구성 템플릿 (2–3시간 권장)
1. **오리엔테이션(10–15분)**: 오늘의 목표, 핵심 개념 소개
2. **이론 미니강의(20–30분)**: 개념/원리/프레임워크 핵심만 요약
3. **데모(15–25분)**: 실제 사례를 빠르게 시연
4. **실습(60–90분)**: 개인 또는 조별 과제 수행
5. **공유(15–25분)**: 결과 발표, 상호 피드백
6. **과제 안내(5–10분)**: 다음 주 적용 과제 제시

# 주차별 커리큘럼

## Week 1: AI 기초와 리터러시
**목표**: AI가 무엇이며 왜 쓰이는지, 한계와 위험을 이해한다.

- **핵심 개념**: 생성형 AI, 모델/데이터 개념, 환각/편향, 프롬프트 기본
- **이론 강화**: AI 시스템 구성요소(데이터–모델–앱–업무), 학습 vs 추론, 신뢰도/평가, 한계/리스크
- **실습**:
  - 내 업무/일상에서 AI 적용 가능한 과제 3가지 도출
  - 동일 질문을 2개 도구에 비교 실행해 차이 기록
- **과제/산출물**:
  - “나의 AI 활용 목표 선언문(1쪽)”
- **준비/자료**:
  - 샘플 프롬프트 카드(아래 템플릿)

**프롬프트 카드 템플릿**
- 역할: (예: 마케팅 담당자)
- 목표: (예: 홍보 문구 3안 만들기)
- 제약: (예: 200자 이내, 특정 금지어)
- 검증: (예: 정확성/출처 확인)

## Week 2: 실무 프롬프트 & 업무 자동화 기초
**목표**: 실무 문서·메일·회의록을 AI로 처리하는 기본 패턴을 익힌다.

- **핵심 개념**: 프롬프트 패턴(역할/목표/제약/검증), 요약/작성/번역 워크플로우
- **이론 강화**: 프롬프트=요구사항 명세, 품질 기준(정확성/일관성/재현성), 간단한 평가 루프
- **실습**:
  - 긴 글 → 요약 → 액션아이템 추출
  - 회의록 초안 자동 생성
- **과제/산출물**:
  - 실제 업무 문서 1건을 AI로 개선하여 제출
- **준비/자료**:
  - 샘플 회의록 템플릿

**회의록 템플릿**
- 목적/의제:
- 핵심 논의:
- 결정 사항:
- 담당자/기한:
- 이슈/리스크:

## Week 3: 데이터/문서/스프레드시트 활용
**목표**: 표 데이터 이해 및 간단한 분석 요청 방법을 익힌다.

- **핵심 개념**: 표 구조 이해, 필터/집계, 데이터 해석 문장 만들기
- **이론 강화**: 데이터 품질(정확성/적시성), 데이터 흐름, 지표–의사결정 연결
- **실습**:
  - 매출/고객 샘플 데이터에서 인사이트 3개 도출
  - 데이터 요약 → 한 페이지 리포트 작성
- **과제/산출물**:
  - “데이터 인사이트 1페이지”
- **준비/자료**:
  - 샘플 CSV/시트

**데이터 인사이트 작성 프롬프트 예시**
- 목표: 주간 매출 변동 원인 3가지 추정
- 제약: 수치 근거 포함, 5문장 이내

## Week 4: 노코드/자동화 워크플로우
**목표**: 반복 업무를 자동화 흐름으로 설계한다.

- **핵심 개념**: 트리거/액션/조건, 자동화 설계 원칙
- **이론 강화**: AS-IS/TO-BE 프로세스 맵, 자동화 우선순위(Impact/Effort), 예외 흐름 설계
- **실습**:
  - 폼 → 시트 → 알림 자동화 설계도 작성
  - 업무 프로세스 1개를 자동화 흐름으로 재구성
- **과제/산출물**:
  - 자동화 워크플로우 1장(그림/표)
- **준비/자료**:
  - 워크플로우 템플릿

## Week 5: AX 컨설팅 기초
**목표**: 문제정의, 현업 인터뷰, ROI/리스크 관점의 사고를 익힌다.

- **핵심 개념**: 문제정의, 사용자/이해관계자 분석, ROI, 변화관리
- **이론 강화**: 문제정의 → 이슈 트리 → 가설 → KPI/ROI 연결, 이해관계자 관점
- **실습**:

## Week 6: 캡스톤 — AX 컨설팅 제안
**목표**: 실제 업무 문제를 AX 관점으로 해결하는 미니 제안서를 완성한다.

- **결과물**: 3–5페이지 제안서 + 5분 발표 + 아키텍처 스케치 1장
- **이론/설계 강조**: 간단 아키텍처 스케치(입력–처리–출력, 데이터 흐름, 연동 포인트), 운영/리스크
- **구성 권장안**:
  1. 배경/문제 정의
  2. 현행 프로세스 요약
  3. 목표/성공지표
  4. 제안 솔루션(업무+AI)
  5. 실행 로드맵(0–3개월)
  6. ROI/리스크/변화관리

### 평가 루브릭 (각 1–5점)
| 항목 | 설명 |
| --- | --- |
| 문제 정의 | 현황과 문제의 구체성/명료성 |
| 해결안 적합성 | 제안이 문제와 맞는지, 실행 가능성 |
| 데이터/근거 | 최소한의 수치/근거 제시 여부 |
| 아키텍처/설계 | 구성의 논리성, 데이터 흐름/연동 고려 |
| ROI/리스크 | 기대효과와 위험요소의 균형 |
| 변화관리 | 조직·사용자 수용 전략 포함 여부 |
| 발표/문서 | 전달력, 구조화, 가독성 |
